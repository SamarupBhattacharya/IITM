# TDS Virtual TA — Course Q&A API

A Retrieval-Augmented Generation (RAG) system that answers student questions from IIT Madras's **Tools in Data Science (TDS)** course. It scrapes and indexes the course's Discourse doubt-solving channel, then exposes a FastAPI endpoint that accepts text questions — optionally with images — and returns a grounded answer alongside the most relevant source link.

## Overview

Students on IIT Madras's online degree programme ask hundreds of questions on a Discourse forum. This project turns that forum into a searchable, AI-powered knowledge base:

1. Posts are scraped and cleaned from the forum.
2. Images embedded in posts are described by a vision model (Gemini 2.0 Flash) so their content is searchable as text.
3. All content is chunked and embedded with OpenAI's `text-embedding-3-small` model.
4. At query time a FastAPI server embeds the incoming question, retrieves the top-5 most similar chunks via cosine similarity, and asks GPT-4o-mini to synthesise a concise, grounded answer.
5. If the question itself includes an image, Gemini Vision describes it first and that description is folded into the query.

## Pipeline

### Step 1 — Discourse Scraping

**Script:** `discourse_scraping.py`

Authenticates to `https://discourse.onlinedegree.iitm.ac.in` using browser session cookies and scrapes all topics from **category 34 (TDS-KB)** created between `2025-01-01` and `2025-05-31`.

For every topic it:
- Paginates through all posts using the Discourse JSON API.
- Strips HTML with BeautifulSoup to extract clean plain text.
- Collects all `<img src="...">` URLs into a separate list.

Outputs:
- `raw_topic_posts.json` — full Discourse API JSON for every topic page.
- `processed_posts.json` — list-of-lists, each inner list representing one topic's posts as `{ "text": "...", "image_url": [...] }`.

> **Note:** The session cookies in the script are examples. Replace `_t` and `_forum_session` with your own active browser cookies before running.

---

### Step 2 — Image Explanation

**Script:** `image_explainer.py`

Reads `processed_posts.json` and replaces every image URL with a detailed textual description generated by **Gemini 2.0 Flash**.

For each URL it:
1. Downloads the image (skips images > 7 MB or with unsupported MIME types).
2. Base64-encodes the content.
3. Sends it to the Gemini API with the prompt *"Explain and caption this image in detail."*
4. Stores the returned description in place of the original URL.

Outputs:
- `processed_posts_with_explanations.json` — same structure as `processed_posts.json` but with image descriptions instead of URLs.

> **Requires:** `GEMINI_API_KEY` set in the script (or as an environment variable).

---

### Step 3 — Markdown Conversion

**Script:** `discourse_jsons_to_md.py`

Reads `processed_posts_with_exp.json` (the explained version) and writes one Markdown file per topic batch into the `discourse_markdowns/` directory.

Each file (`d1.md`, `d2.md`, …) contains the plain text of every post in that topic, with image explanations inlined as text, ready for chunking.

---

### Step 4 — Embedding Generation

**Script:** `embedding_generation.py`

Scans the working directory for all `.md` files and builds the vector knowledge base.

1. Splits each file using `RecursiveCharacterTextSplitter` (chunk size 2 000 chars, 20% overlap) with Markdown-aware separators.
2. Sends chunks to **OpenAI `text-embedding-3-small`** in batches of 50 via the direct OpenAI API.
3. Saves all chunks and their 1 536-dimensional embeddings to `course_knowledge_base.npz`.

> The production file committed to the repository is `course_with_id.npz`, which additionally stores the source Discourse URL alongside each chunk so the API can return direct links.

---

### Step 5 — Serving the API

**Script:** `main.py`

A **FastAPI** application that loads `course_with_id.npz` at startup and serves a single `POST /` endpoint.

**Request flow:**

1. If an image is provided, Gemini Vision generates a ~50-word description.
2. The description is appended to the question to form a combined query string.
3. The query is embedded with `text-embedding-3-small` via the **AI Proxy**.
4. Cosine similarity is computed against all stored embeddings; the top 5 chunks (and their URLs) are retrieved.
5. GPT-4o-mini is prompted with the retrieved context and asked for a concise, grounded answer.
6. The answer and the URL of the most similar chunk are returned.

## API Reference

### `POST /`

**Request body (JSON):**

| Field | Type | Required | Description |
|---|---|---|---|
| `question` | `string` | ✅ | The student's question in plain text. |
| `image` | `string` | ❌ | Base64-encoded image (WebP preferred; JPEG and PNG also supported). May include a `data:image/...;base64,` prefix. |

**Response body (JSON):**

| Field | Type | Description |
|---|---|---|
| `answer` | `string` | Concise answer grounded in course materials. |
| `links` | `array` | List of `{ "url": "...", "text": "..." }` objects — the most relevant Discourse post. |

**Example request:**

```bash
curl -X POST https://<your-deployment-url>/ \
  -H "Content-Type: application/json" \
  -d '{
    "question": "What is the difference between RMSE and MAE?",
    "image": null
  }'
```

**Example response:**

```json
{
  "answer": "RMSE penalises large errors more heavily because it squares the residuals before averaging, whereas MAE treats all errors equally by taking their absolute values. Use RMSE when large errors are especially undesirable; use MAE for a more robust, interpretable metric.",
  "links": [
    {
      "url": "https://discourse.onlinedegree.iitm.ac.in/t/some-topic/12345",
      "text": "RMSE vs MAE — when to use which metric..."
    }
  ]
}
```

## Setup & Installation

```bash
# 1. Clone the repository
git clone <repo-url>
cd <repo-directory>

# 2. Create and activate a virtual environment (Python 3.11+ recommended)
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt
```

### Running the pipeline from scratch

```bash
# Step 1 — scrape forum (update cookies in script first)
python discourse_scraping.py

# Step 2 — explain images
python image_explainer.py

# Step 3 — convert to Markdown
python discourse_jsons_to_md.py

# Step 4 — generate embeddings
cd discourse_markdowns
python ../embedding_generation.py

# Step 5 — start the API server
cd ..
uvicorn main:app --host 0.0.0.0 --port 8000
```

If you already have `course_with_id.npz`, you can skip straight to Step 5.

## Environment Variables

| Variable | Used by | Description |
|---|---|---|
| `OPENAI_API_KEY` | `main.py`, `embedding_generation.py` | OpenAI API key (used via AI Proxy in `main.py`). |
| `GOOGLE_API_KEY` | `main.py` | Google Gemini API key for vision queries at runtime. |
| `GEMINI_API_KEY` | `image_explainer.py` | Google Gemini API key for offline image captioning. |

Set them before running:

```bash
export OPENAI_API_KEY="sk-..."
export GOOGLE_API_KEY="AIza..."
export GEMINI_API_KEY="AIza..."
```
